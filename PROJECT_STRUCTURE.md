# AgriGo - Multi-Module Project Structure

## Project Overview

This is a multi-module project containing:
- **server**: Backend API and services (Python/Node.js)
  - `backend/`: Flask-based REST API
  - `agrinova-ui-main/`: Web UI (React/TypeScript with Vite)
- **client**: Native Android application (Kotlin/Compose)

## Directory Structure

```
AgriGo/
├── server/              # Backend services and web UI
│   ├── backend/         # Python Flask API
│   ├── agrinova-ui-main/  # Web UI (React)
│   └── ...
├── client/              # Android app (Gradle multi-module)
│   ├── app/             # Main Android application
│   ├── gradle/          # Gradle configuration and version catalogs
│   └── ...
├── build.gradle.kts     # Root Gradle build file
├── settings.gradle.kts  # Root Gradle settings (multi-module configuration)
├── gradlew.bat          # Gradle wrapper for root directory
└── local.properties     # Android SDK configuration
```

## Building the Project

### From Root Directory

You can now build the Android app from the root directory:

```powershell
cd C:\Users\aksha\AndroidStudioProjects\AgriGo

# Build debug APK
.\gradlew.bat :client:app:assembleDebug

# Build release APK
.\gradlew.bat :client:app:assembleRelease

# Run tests
.\gradlew.bat :client:app:test

# List all available tasks
.\gradlew.bat :client:app:tasks

# Run linting
.\gradlew.bat :client:app:lint
```

### Alternative: From Client Directory

You can also build from the client directory directly:

```powershell
cd client
.\gradlew.bat app:assembleDebug
```

## Multi-Module Configuration

### Root Settings (`settings.gradle.kts`)

The root settings file configures:
- **Plugin Management**: Manages gradle plugin repositories
- **Dependency Resolution**: Configures artifact repositories (Google, Maven Central)
- **Version Catalogs**: Loads the shared version catalog from `client/gradle/libs.versions.toml`
- **Project Includes**: Includes `:client` and `:client:app` modules

### Client Settings (`client/settings.gradle.kts`)

Provides plugin management and dependency resolution for the client module specifically.

### Root Build (`build.gradle.kts`)

Provides common build configuration for all subprojects.

## Version Catalog

The version catalog is located at: `client/gradle/libs.versions.toml`

This file defines:
- **Versions**: Android AGP, Kotlin, libraries versions
- **Libraries**: Dependency declarations
- **Plugins**: Plugin declarations

All modules automatically have access to these definitions via the `libs` namespace.

## Local Configuration

### local.properties

Contains local machine-specific settings:
```ini
sdk.dir=C:\\Users\\[USERNAME]\\AppData\\Local\\Android\\Sdk
```

This file is automatically generated and should NOT be committed to version control.

## Gradle Tasks Reference

Common tasks you'll use:

### Development Tasks
- `assembleDebug` - Build debug APK
- `installDebug` - Install debug APK on connected device
- `compileDebugKotlin` - Compile Kotlin code

### Testing Tasks
- `test` - Run unit tests
- `connectedAndroidTest` - Run instrumented tests on device
- `lint` - Run Android Lint analysis

### Build Tasks
- `assembleRelease` - Build release APK
- `bundleRelease` - Build app bundle (for Google Play)

### Clean Tasks
- `clean` - Clean build directory
- `cleanBuildCache` - Clear Gradle build cache

## Environment Setup

### Prerequisites
- Android Studio (or command-line tools)
- Java 21+ (for Gradle 9.3.1)
- Android SDK (minimum API 30)

### Initial Setup
1. Ensure `ANDROID_HOME` environment variable is set or `local.properties` has correct `sdk.dir`
2. Run: `.\gradlew.bat :client:app:assembleDebug`

## Troubleshooting

### "SDK location not found"
Ensure `local.properties` in root directory has correct Android SDK path:
```ini
sdk.dir=C:\\path\\to\\Android\\Sdk
```

### "Settings file not found"
Make sure you're running from the root directory: `C:\Users\aksha\AndroidStudioProjects\AgriGo`

### "Project 'app' not found"
This happens if paths in `settings.gradle.kts` are incorrect. Verify project directories exist at:
- `client/`
- `client/app/`
